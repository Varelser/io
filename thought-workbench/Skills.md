# Skills

このファイルは、Thought Workbench に実装するべき能力と、その実装方針を定義する。

## 1. コア原則

### 1.1 主軸
- 主軸は「思考を空間化し、深め、再構成すること」
- 単なるメモ保管や文献倉庫に寄りすぎない
- タスク管理や分類法は補助層として扱う

### 1.2 基本思想
- 1トピック = 1球体
- ノードは最小単位の思考片
- 線は単なる接続ではなく、意味を持つ
- 時間、層、重要度、反対意見を同一環境で扱う
- ユーザーが自由に壊せる余地を残す

### 1.3 UI 原則
- 黒背景 / 白文字 / ミニマル
- 中央の球体表示を最優先
- 左右パネルは補助
- テキストは小さく、情報密度を高く
- 折りたたみ可能にする

## 3. プリセットごとの技能定義

## 3.1 Free
### 目的
- 制約なく思考を配置する

### 必要技能
- 自由配置
- 自由接続
- 最小限のテンプレート

## 3.2 Mandala
### 目的
- 中心テーマから 8 方向に展開する

### 必要技能
- 9 マス初期構造生成
- 中心 / 周辺テンプレート
- 中心切替時の再配置

## 3.3 Semantic
### 目的
- 概念と関係を明示する

### 必要技能
- cause / effect / example / counter / concept テンプレート
- 階層的再配置
- 関係語付き接続

## 3.4 Strata
### 目的
- 深度・層・堆積として考える

### 必要技能
- surface / middle / deep / archive テンプレート
- Z 軸優先配置
- layer 可視化

## 3.5 Zettelkasten
### 目的
- 原子ノードとリンク理由を重視し、連鎖的に考える

### 必要技能
- question / claim / evidence / counter テンプレート
- orphan 検出
- backlink 強調
- 接続理由テンプレート

## 3.6 PARA
### 目的
- topic 群を上位整理する

### 必要技能
- Projects / Areas / Resources / Archives の分類
- Workspace 上でのカテゴリ配置
- topic 単位の一括整理

## 3.7 GTD
### 目的
- 実行可能な行動へ落とす

### 必要技能
- inbox / next / waiting / someday / review
- 実行ビュー
- review 補助

## 3.8 Must One
### 目的
- 今日の最重要ノードを固定する

### 必要技能
- 固定表示
- 強調表示
- 日別管理（将来）

## 4. 追加実装ルール

### 4.1 機能追加時
- 既存操作を壊さない
- データ構造を不用意に変えない
- JSON 互換性をできるだけ保つ
- ユーザー未指定の大規模拡張は避ける

### 4.2 UI 変更時
- 文字は小さく
- 情報は折りたたむ
- 中央表示の面積を減らさない

### 4.3 パフォーマンス
- 無駄な再計算を減らす
- ラベル表示は必要条件付き
- 大量ノード時の簡易表示モードを将来追加

## 6. 完成条件

このプロジェクトの「最低限の完成」は以下を満たすこと。

- 空間配置が安定して行える
- 時間と履歴を扱える
- プリセットが方法論として機能する
- 思考の深まりを支援できる
- 自分専用ツールとして毎日使える
